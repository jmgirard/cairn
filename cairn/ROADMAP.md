## Candidates

- Work-log staleness signal is corrupted by amendment entries: `/milestone` §2 flags an `in-progress` milestone with "no work-log entry in 14+ days", but a work log accumulates entries that record *queueing* rather than work — `Depends-on` amendments, mirror catch-ups, status transitions — and each refreshes the clock, so a genuinely stalled milestone reads as active. Found live at M88 T3: intraclass M48 had sat `planned` and unworked for a week while every one of its entries was a `Depends-on` amendment, which is what forced M88-D1 to abandon idleness for the `planned` case entirely. M88 solved it only for release-shaped milestones; the general signal is still wrong for every other milestone the audit judges. Fix direction: classify entries (work vs. bookkeeping) or measure from the first entry at a status rather than the last. Swept: no row, archive entry, or D-entry covers work-log signal quality — M83 is references-extraction staleness, a different parser — added 2026-07-19 — M88 T3
- Prior-PR-comments review lens is a standing no-op in a solo-maintained repo: `/milestone-review`'s lens reads GitHub PR review threads, but ackwards M68, M67, and M61 each logged zero findings from it — M67 recorded its touched files as "changed by direct-to-master commits" with no PR comments, M61 logged "prior-PR: no evidence (no-op)" — because the substantive review record lives in archived milestone `## Review` sections while the PR threads hold only Codecov bot noise. It spends `gh api` calls and a reasoning pass at every review looking where the content provably is not, and three consecutive clean results are indistinguishable from a lens that cannot fire. Fix direction: point it at `milestones/archive/` `## Review` sections, which M77's lesson already names as the home of milestone-local decisions; needs a decision on whether the PR-thread read is replaced or kept as a second surface for repos that do review on GitHub. Swept: no candidate row, archive entry, or D-entry covers the lens's input surface — added 2026-07-19 — M89 plan (field evidence, ackwards)
