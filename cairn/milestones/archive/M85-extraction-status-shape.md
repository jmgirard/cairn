# M85 — Extraction-status shape: the templates teach what the classifier reads

**Status:** done · approved 2026-07-18 · PR https://github.com/jmgirard/cairn/pull/84

**Goal.** Make both `references/` templates teach the *shape* of an extraction status the staleness classifier can read, and prove each sanctioned form classifies as its author intends.

**Outcome.** Both templates now state the rule the classifier applies — a status must claim a verification, carry a date, or say there is nothing to re-verify — name the four affirmative verbs plus `unverified` as the self-negating exception, and say the listed alternatives are examples of that shape, not the accepted list. Ten new tests select each `<a | b | c>` alternative in turn and assert the real `_last_verified` verdict against an INTENT table, so an added alternative with no declared meaning fails rather than passing unexamined; the four forms this repo writes but neither template lists are pinned the same way, each checked to be still written by a committed page so the list cannot go fictional. Skills 374 (was 363).

**Origin and the reversal.** Promoted from the "References note-template vocabulary" candidate, whose framing was to add the four unlisted phrases to the templates. The plan gate rejected that: M83's classifier reads a shape, not a whitelist, so an enumeration would be wrong at the fifth phrasing. Investigation then found a live gap the row never named — M80's pairing test instantiated dates but left the `<a | b | c>` choice unmade and asked only existence questions, so unchosen the source template classified `ambiguous` and the synthesis template `exempt`. The templates were correct by luck, not by test.

**Review.** 3 lenses + scorer. Blame-history and prior-PR: no findings (the latter confirmed no inline PR-comment evidence exists for these files across 11 merged PRs). Diff-bug lens: 3 findings, all reproduced independently and all fixed on the branch. F1/93 — the new verb-set sentence added a second `unverified` to `source-note.md`, degrading the pre-existing `test_unverified_extraction_is_an_expressible_state` (a bare whole-file `assertIn`) into false coverage: the alternative could be deleted and the guard stayed green, where on `main` it failed; re-anchored on the `Extraction:` field line. F2/80 — both templates taught a rule the classifier does not implement, listing `unverified` among the affirmative verbs while `_UNVERIFIED` classifies `never` unconditionally, an error already pinned by a guard and two registry entries; stated as an exception and separately guarded. F3/62 — sub-threshold but fixed: an exact-equality test let any shortened variant slip the premise its docstring claimed. Each fix falsified by mutation after landing. Nothing logged-and-dropped.

**Note for adopters.** `TestUnlistedShippedFormsSatisfyTheShapeRule` asserts against this repo's own `cairn/references/` pages — a plugin test reading repo content (precedent: `TestBackfilledPages`). Raised at the merge gate and accepted; it is the seam most likely to behave differently in an adopting repo.
