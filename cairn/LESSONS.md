# Lessons

Durable repo lessons — build quirks, testing tricks, gotchas worth
remembering next time — captured at milestone end and surfaced at plan time.
Not status, not decisions: a lesson is a reusable "how this repo actually
behaves" note. Cross-cutting *choices* still go to `DECISIONS.md`.

One line per lesson: `- YYYY-MM-DD (M<NN>): <lesson>`. One cap
(tracking-rules weight-caps): 50 lines, met by retiring or pruning entries.
Current knowledge: a lesson proven false is corrected in place (D-045).
Lessons also *leave*: one retires when a test fails on the mistake it warns
about, when another file's slot owns its content, or when a matured family
graduates whole into a doctrine module; pruning the stalest is the last resort
rather than the first (D-051, D-055).

- 2026-07-12 (M35): when a milestone's scope was drawn from existing ROADMAP candidate rows, those rows are NOT pruned at plan time (candidates graduate at completion, not at plan) — graduate them in the post-merge hygiene pass or the ROADMAP advertises shipped work as pending; both review lenses flagged M35's two fulfilled rows.
- 2026-07-12 (M51, trimmed M99): trimming prose only reduces a line count when it removes a whole *wrapped* physical line — a shorter sentence that still wraps to N lines saves nothing, so cut content, don't rephrase. (The ≤25 archive cap, its per-section allocation, and counting as you draft now live in `templates/archive-summary.md`, `/milestone-review` step 9, and `cairn_budget`.)
- 2026-07-16 (M56+M65, consolidated M78/M83): "green" is only as wide as what you ran. Bash's working directory persists across tool calls and piping a verify through `| tail -1` makes the chain exit-blind (`tail` exits 0 even when `discover` ImportErrors) — a lingering `cd cairn` silently skipped all three suites, and at the M65 gate the pipe hid a red exit. Run suites from the repo root, check each exit code explicitly, and never chain a commit/push/marker write after a piped run. And `cairn_validate` green is NOT the suites green: a `cairn/references/` edit passing every validate check can still break a `skills/tests` guard, and a trivial-tier docs commit pushed main red because only validate was run — any references or template edit runs the three suites too, trivial tier or not.
- 2026-07-16 (M60): refines M19 — hook *registrations* (hooks.json) snapshot at process start, but each script BODY is re-read per invocation from the symlinked checkout: editing an already-registered hook goes live mid-session (the new merge_guard renamed the marker during M60's own merge), while a newly-registered hook stays dead until a fresh conversation; a fixture test still proves only what a hook prints, and live-fire plans must split the two cases.
- 2026-07-16 (M60/M85, trimmed M98): a mutation-registerable target is not only a test's own prose — TEMPLATE classes CAN be registered (M85 added ten), so a new guard over a template needs its own `Mutation(...)` entry; the completeness meta-test reddens on an unregistered guard *file* and will not catch the omission.
- 2026-07-16 (M61): don't bulk-edit tracking files with ad-hoc string scripts — a review-time `str.partition` tick-the-boxes script silently truncated the milestone file at the section heading (only `git checkout --` recovered it); tick checkboxes with targeted Edit replacements, and if a script must touch a tracking file, `wc -l`/diff it before moving on.
- 2026-07-17 (M69): when implementation improves on a planned output token/format (here `+N over` → the more actionable `shed ≥N`), amend the AC via the implement step-6 gate in that same task — don't let plan text drift from the delivered string, or review's "never reinterpret the AC" rule correctly fails it (two independent lenses caught it here).
- 2026-07-18 (M71, corrected M75, consolidated M92): hooks trap on MATCHING and on SHIPPING. Matching: a `matcher` made only of `[a-zA-Z0-9_|]` (or `[a-zA-Z0-9_|, -]`) takes a LITERAL path — split on `|`/`,`, each alternative exact-matched — so `|` does NOT buy regex treatment; only a metacharacter (`mcp__.*__spawn_task`) reaches `new RegExp`, and that path is unanchored, so keep the hook's own tool-name regex the same suffix shape (full dispatch: `references/claude-code-hooks.md`). Shipping: a new hook touches four places beyond the script itself — `hooks.json`, BOTH `TestNonCairnNoOp` payload collections in `hooks/tests/test_hooks.py` (dict + garbage-stdin tuple), `cairn/DESIGN.md`'s inventory bullet (count words included), and `test_positioning_guard.py`'s hardcoded `HOOKS` tuple, NOT derived from `hooks/`.
- 2026-07-18 (M72, extends M33, trimmed M92/M98): the merge guard is direction-blind — `git merge main` INTO a feature branch is denied though tracking-rules prescribes merging when main moves, so sync a feature branch with `git rebase main` instead.
- 2026-07-18 (M73): a sub-threshold confidence score gates the ACTIONED list, not the operator's judgment — the scorer dropped "the fork fallback closes a contributor's PR with no approval gate" at 48 on a false equivalence (opening your own PR is reversible; closing someone else's is not). Read every sub-80 finding's substance, and treat any finding that authorizes an outward-facing irreversible action as worth fixing regardless of score. **Extended M88:** the scorer's *reasoning* can be wrong while its number is defensible — it refuted a token-gap finding by crediting a token that did match, when the case actually failed on a missing version; verify a refutation against the implementation before accepting it.
- 2026-07-18 (M73): `gh pr create`/`ready`/`merge` all use GraphQL, whose quota can be exhausted while REST is healthy — `gh api repos/{owner}/{repo}/pulls --input <json>` opens a PR (draft included) when `gh pr create` is rate-limited. Never route the MERGE through REST to dodge a limit: merge_guard is a PreToolUse hook matching the merge commands, so a REST merge silently bypasses the approval marker — the enforcement boundary M72 documented.
- 2026-07-18 (M77): the plan-time collision sweep must grep `milestones/archive/` for *decisions*, not just DECISIONS.md and candidate rows — M69 recorded "work log stays counted" as a milestone-local decision with no D-entry, and M77 reversed it without citing it; a milestone-local decision is superseded in the same milestone-local form, since DECISIONS.md is history IP4 forbids editing.
- 2026-07-18 (M78): a milestone that ships a records-hygiene rule must run that rule over the artifacts the milestone ITSELF authors — M78's 16-page provenance backfill introduced 16 undated repo-state claims, verbatim the category its own new rule forbids; the compliance fix committed the failure it was fixing, and the diff-bug lens caught it, not the author (F3/90).

- 2026-07-19 (M87): at review the cap breakdown can name a section review is forbidden to touch — M87's overrun ranked plan-owned `Scope` heaviest while the growth was review's own evidence in `## Decisions`. - 2026-07-19 (M90): `Edit` with `replace_all` applies ONE fixed indentation to every match — in Python that silently changes block structure and stays valid. A nested loop dedented to its parent's level ran once after the parent finished, against the last-bound variable, so a two-surface guard checked one surface and passed green (F1/96). - 2026-07-20 (M99): a figure stated INSIDE the artifact it measures is a fixed point — writing it changes it. The milestone template's budget block stated its own preamble length and drifted twice in one authoring session under full attention; the fix was not a better guard but DELETING the figure, since the claim worth pinning was that the budgets fit, which a guard asserts against a measured value. Prefer a tool that prints the number over prose that states it.
- 2026-07-20 (M99): rewriting an instruction from "transform X into Y" to "author a fresh Y" silently drops X's disposal — step 9 said "compress the milestone file and move it", where the move deleted the live copy implicitly; authoring from a template made the summary a NEW file and nothing removed the old one, orphaning it until `cairn_validate` caught it (F4/85). When a rewrite changes what an artifact IS, re-derive every side effect the old phrasing carried implicitly.
- 2026-07-20 (M95): the mutation harness catches an anchor whose prose REFLOWED, but nothing catches an anchor authored against text that was never read out — three asserts in a new guard failed at once on wrap points invented from the draft, and a `[:34]` slice hack nearly pinned a truncated clause. Author anchors by copying the target file's actual bytes, never from the text you just wrote.
- 2026-07-20 (M95): an inversion sweep's probe locators must be drawn from the GUARD files, not from the prose being cut — a locator that misses reads exactly like a rule that is unguarded, and two rules were reported unpinned on phrases (the elaboration, not the rule) that no guard had ever asserted. - 2026-07-20 (M99): a measured distribution jammed against a cap measures the CAP, not demand — 55 of 96 archive summaries sat at exactly 25 lines, so their percentiles were censored and could not be a budget basis, while the milestone-body population (spread 97–149) could. Check for a spike at the limit before deriving a budget from artifacts that limit already governs; when censored, set the budget deliberately BELOW the median.

- 2026-07-20 (M100): an enforcement check must fail LOUD, never open — review found four paths where the binding-criteria check silently bound nothing (HTML comments counted in the scanned corpus, a present-but-unparseable section skipped, an id-prefix marker match, a malformed slot skipped).
